# PH Generator — 技术实现说明书

## 整体架构

### 技术选型及理由

| 技术 | 用途 | 选择理由 |
|------|------|----------|
| Godot 4.6 | 运行平台 + 编辑器 | 目标用户的日常工作环境 |
| GDScript | 全部逻辑 | Godot 原生语言，零编译，热重载 |
| EditorPlugin API | 编辑器集成 | 嵌入编辑界面，无需切换工具 |
| HTTPRequest | LLM 通信 | Godot 内置 HTTP 客户端 |
| SurfaceTool / ArrayMesh | 程序化建模 | Godot 核心 API，生成 .glb/.fbx |
| OpenAI 兼容 API | AI 解析 | 标准化接口，可切换多种模型 |
| Python 3 + trimesh | FBX 导出 | glTF→FBX 转换，轻量级依赖 |

## 关键实现

### 1. LLM 交互协议（ai/）

**llm_client.gd**  
封装 HTTP POST 请求。配置 OpenAI 兼容接口格式：
```json
{
  "model": "gpt-4o",
  "messages": [...],
  "temperature": 0.3,
  "response_format": {"type": "json_object"}
}
```

**prompt_mgr.gd**  
System Prompt 设计要点：
- 定义可用 primitive 类型（box/cylinder/sphere/capsule/wedge）
- 定义材质-颜色映射表
- 内置 5 个 Few-shot 示例（车/箱/掩体/门/柱）
- 输出格式严格约束为 JSON

**parser.gd**  
对 LLM 返回做鲁棒解析：
1. 首选 `JSON.parse_string` 直接解析
2. 失败时搜索 `primitives` 关键字，提取平衡括号内的 JSON
3. 仍未成功则构建默认 box fallback

### 2. 程序化网格生成（core/mesh_builder.gd）

基于 `SurfaceTool` API 手动构建顶点数据：

**Box**: 8 个顶点 → 6 个面（12 三角形）
**Cylinder**: 圆周分段 + 顶底三角形扇
**Sphere**: 经纬度采样 → 三角形网格
**Capsule**: 上半球 + 柱体 + 下半球，圆周分段
**Wedge**: 5 个顶点 → 三角棱柱（半个 Box）

关键代码模式：
```gdscript
func build_box(data: Dictionary) -> ArrayMesh:
    var st = SurfaceTool.new()
    st.begin(Mesh.PRIMITIVE_TRIANGLES)
    # add vertices...
    st.generate_normals()
    return st.commit()
```

### 3. 场景构建（core/scene_builder.gd）

为每个 primitive 创建独立 MeshInstance3D 并挂载到 PH 根节点：
- 材质使用 `StandardMaterial3D`，按类型设置 `roughness`/`metallic`
- 自动添加 `Label3D` 显示物件名称
- 自动附加 `StaticBody3D` + `CollisionShape3D`

### 4. 离线预设系统（presets/library.gd）

**关键词匹配**: 从描述文本中匹配预设关键词（中文+英文），选择最长匹配项。  
**尺寸缩放**: 将预设模板等比例缩放至目标白盒尺寸：
```
scale_ratio = target_dim / preset_bounding_box
```
对所有 primitive 的 size/position/radius 做等比缩放。

内置 10 类预设：车、箱子、掩体、门、柱子、楼梯、窗户、沙发、桌子、椅子。

### 5. 导出（export/）

**gltf_exporter.gd**: 使用 `GLTFDocument.append_from_scene()` → `write_to_filesystem()`

**glb2fbx.py**: 三级降级策略
1. trimesh（纯 Python，最轻量）:`scene = trimesh.load(src); scene.export(dst)`
2. assimp CLI: `assimp export src dst`
3. Blender headless: `bpy.ops.import_scene.gltf` → `export_scene.fbx`

## 使用的 AI 工具

| 环节 | AI 工具 | 作用 |
|------|---------|------|
| 需求分析 | opencode (DeepSeek-V4) | 拆解关卡策划工作流痛点 |
| 架构设计 | opencode | 模块划分与技术选型 |
| 代码实现 | opencode | 全部 GDScript + Python 代码 |
| 运行时 AI | LLM API（OpenAI 兼容） | 自然语言 → 结构化几何体 |
