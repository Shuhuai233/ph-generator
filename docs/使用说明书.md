# PH Generator — 使用说明书

## 适用场景

针对**关卡策划（LD）白盒 → 场景美术（LA）PH 建模**这个纯体力环节，AI 自动将白盒尺寸 + 物件描述转为 3D 占位模型（PH），省去手工建模的重复劳动。

**典型工作流：**
1. LD 在场景中搭好白盒（BoxMesh 原始体块）
2. LA 选中白盒，输入物件描述（如"轿车 4×1.8×1.5"）
3. 工具调用 LLM 解析描述 → 程序化生成 PH
4. 一键导出 .glb / .fbx 交付

## 快速开始

### 1. 安装

将 `addons/ph_generator/` 文件夹放入你的 Godot 4.6 项目，插件会自动启用。  
（插件已在 `project.godot` 中注册）

### 2. 配置 API Key

1. 打开 Godot → 右侧 Dock 面板找「PH Generator」
2. 点击「⚙ 设置」按钮
3. 填写：
   - **Endpoint**: LLM API 地址（默认 OpenAI `https://api.openai.com/v1/chat/completions`）
   - **API Key**: 你的密钥
   - **Model**: 模型名称（如 `gpt-4o`、`deepseek-chat` 等）
   - 点击「保存」

> 支持任何 OpenAI 兼容 API（DeepSeek, Claude via API proxy, GLM 等）

### 3. 基本使用

**方式一：自由描述**
1. 在「物件描述」框中输入，如：`轿车 (4×1.8×1.5)`
2. 下方调整「尺寸 X/Y/Z」数值
3. 点击「AI 生成 PH」
4. 等待 2-5 秒，PH 出现在场景视口

**方式二：从选中白盒读取**
1. 在场景树选中一个 MeshInstance3D
2. 勾选「从选中节点读取尺寸」
3. 输入物件描述
4. 点击「AI 生成 PH」

**方式三：离线快速模板**
1. 下拉选择合适的预设（车/箱子/掩体/门/柱子...）
2. 调整尺寸
3. 点击「AI 生成 PH」（离线模式走模板近似）

### 4. 导出

- **导出 .glb**：默认导出到 `res://exports/` 目录
- **导出 .fbx**：需 Python 3 + trimesh/assimp/blender 其一

## 功能说明

| 功能 | 说明 |
|------|------|
| AI 描述解析 | 自然语言 → 结构化几何体组合 |
| 程序化 PH 生成 | 基于 Box/Cylinder/Sphere/Capsule/Wedge 组合 |
| 材质自动配色 | 按物件类型分配颜色和材质属性 |
| 碰撞体生成 | 自动附加 StaticBody3D + CollisionShape3D |
| 文字标签 | 每个 PH 带 Label3D 描述标签 |
| JSON 预览编辑 | 可查看、手动修改 LLM 解析结果再生成 |
| 预设模板库 | 10+ 常用物件模板，无 API 也能用 |
| 双格式导出 | .glb + .fbx 双输出 |

## 常见问题

**Q: 没有 API Key 能用吗？**  
A: 可以。内置 10+ 预设模板，下拉选择 + 调尺寸即可生成。

**Q: 生成的 PH 能手工修改吗？**  
A: 可以。LLM 返回的 JSON 显示在预览区，修改后点击「生成 PH」即可。

**Q: FBX 导出需要什么环境？**  
A: Python 3 + `pip install trimesh` 即可。如未安装，只导出 .glb。

**Q: 支持哪些 LLM？**  
A: 所有 OpenAI 兼容 API。包括 DeepSeek、Claude（通过 proxy）、GLM、通义千问等。
